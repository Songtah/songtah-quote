# LINE 客情自動抓取原則

本文件說明「LINE 日報 → 客情紀錄」的抓取邏輯，供日後調整規則時對照。
對應程式：
- `app/api/line/webhook/route.ts`（即時 webhook）
- `app/api/line/import/route.ts` + `components/LineImportContent.tsx`（.txt 匯入）
- `lib/line-daily-report.ts`（日報解析）
- `lib/line-txt-parser.ts`（LINE 匯出檔解析）
- `lib/line-salesperson-map.ts`（業務對應表）

---

## 一、兩條抓取管道

| 管道 | 觸發方式 | 用途 |
|------|----------|------|
| 即時 Webhook（`/api/line/webhook`）| LINE 每收到一則新訊息即時呼叫一次 | 日常自動建檔 |
| .txt 匯入（`/admin/line-import`）| 手動上傳 LINE 匯出聊天記錄 | 補抓歷史 / 漏抓 |

兩者共用同一套解析與判斷規則，差別只在資料來源。**沒有排程**；webhook 為事件驅動。

---

## 二、閘門一：發送人必須在「業務名單」

- 用對應表（`lib/line-salesperson-map.ts`）把 LINE 暱稱轉成系統業務名。
- **只有名單上的業務才抓**，不在名單一律跳過（`isKnownSalesperson`）。
- 比對順序：先完全比對 → 再部分包含 → 都沒中才回原名。
- 目前在職名單：

  | LINE 暱稱 | 系統業務名 |

  | Gus | Gus |

> 新增業務時直接改 `lib/line-salesperson-map.ts` 的 `MAP`。

## 三、閘門二：訊息要是「日報格式」

`isDailyReport` 認得 **「行程回報」或「每日報表」** 任一開頭標記，才視為日報。

接著 `parseDailyReport` 決定「客戶清單從哪裡開始」：

| 訊息內容 | 處理 |
|----------|------|
| 有「行程回報」 | 從該行**之後**開始抓（自動排除上方晨間「行程規劃」）|
| 只有「行程規劃」、沒有回報 | 視為晨間規劃，**整則跳過** |
| 兩者皆無（如 Eason：每日報表→職稱→日期→分隔線→1.客戶）| 整則皆為條目，自動略過 header 行 |

---

## 四、客戶條目解析

- **編號條目＝一個客戶**：支援 `1.` `1．` `1、` `1)` `1：`、全形數字，分隔後可有空白。
- **名稱 vs 內容**：`客戶-內容`（破折號前 ≤10 字）或 `客戶，內容`（全形逗號前 ≤8 字）；無分隔符則整行為名稱，內容在後續行。
- **備註**：以 `。` `·` `•` 開頭、或編號條目後的一般文字行，歸入該客戶內容。

**會被剔除的「非客戶」條目：**
- 關鍵字：業務會議 / 內部會議 / 公司會議 / 培訓 / 教育訓練 / 開會 / 進公司 / 例行會議 / 週一會議 / 局寄貨 / 銀行
- 任務動詞開頭：致電 / 通知 / 整理 / 前往 / 協助 / 遠端 / 預約 / 邀約 / 推薦客戶…、時間開頭（9:、14: 等）
- 含 `&`（多任務描述）
- 超過 15 字且不含任何分隔符（疑似整段任務描述）

---

## 五、日期歸屬（非抓取時間窗）

- 優先用日報內「日期：」那行。
- 未寫日期才用預設＝台灣「業務日」：回報窗 17:00～隔日 03:00 算同一天（凌晨 03:00 前發的歸前一天）。
- ⚠️ 此規則只決定「這筆算哪一天」，**不是**「3 點後就不抓」。webhook 任何時間都即時抓。

---

## 六、自動推斷欄位

- **客戶反應**：訂購/成交→積極配合；詢問/有興趣→有興趣；考慮/再看看→需考慮；不需要/已有→暫不需要。
- **是否需追蹤**：內文含 後續/跟進/追蹤/回覆/確認/再聯絡 等 → 標記需追蹤。
- **競品**：掃描內文比對 Notion 競品選項自動填入。

---

## 七、去重與篩選（匯入專用）

- **去重**：同「客戶名稱＋日期」已存在就跳過，不重複建立。
- **篩選業務**：可只匯入指定一位業務。
- **起始日期**：只匯入該日（含）之後的日報。

---

## 八、不會抓的情況（總表）

1. 發送人不在業務名單
2. 訊息無「行程回報／每日報表」標記
3. 只有晨間「行程規劃」
4. 條目被判定為非客戶（會議、任務、純行程）
5. 匯入時：同客戶＋同日期已存在、不符篩選業務、早於起始日期

---

## 附註：Notion API 限流

webhook 與本機 dev 共用同一個 Notion 整合 token。大量匯入（如單一業務上千筆）建議離峰時段或用「起始日期」分批，避免觸發 429 rate limit。
